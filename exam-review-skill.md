# 考试复习全流程

## ⚠️ 第一步：必须先问用户

开始前用 `clarify` 工具逐项确认，全部回答后再执行：

1. **课件/课程资源**：有没有PPT或其他课程材料？路径在哪？
2. **卷子**：有没有往年试卷/模拟卷？路径在哪？
3. **老师划的重点**：有没有？有的话具体是哪些主题或章节？
4. **考试形式**：开卷还是闭卷？

## 第二步：转换课件

1. 用 `search_files(target='files')` 列出课件目录中的所有文件（.pptx / .pdf / .docx）
2. 安装转换工具：`pip install --break-system-packages markitdown pypdf`
3. 对每个课件文件调用 `markitdown` 转换为 .md：

```bash
python3 -m markitdown "课件.pptx" > "课件.md"
```

4. 若源文件是 .docx，直接用 Python 的 zipfile + xml 读取文本（无需额外工具）

## 第三步：整理课件为精讲笔记

1. 逐讲阅读转换后的 .md 文件
2. 提取每讲的核心内容，输出到 `课件笔记_整理/`，规则：
   - 去掉PPT的文字碎片噪音
   - 按逻辑章节组织，而非页码
   - 保留全部技术要点
   - 可自由添加辅助理解的内容
   - 数学公式用 `$$...$$` 或 `$...$` LaTeX 格式（Obsidian兼容）
   - 每讲文件开头写 `> 包含内容：xxx, xxx, xxx` 摘要
3. 用 `terminal` 的 cp 命令复制到最终目录

## 第四步：核查遗漏

1. 对每个文件对（原始转换 vs 整理版）做关键概念比对
2. 重点查 PPT 中偏理论/逻辑的章节是否遗漏（如密码分析、协议缺陷、密钥管理协议）
3. 发现遗漏立即用 `patch` 补全
4. 核查过去真题是否考过遗漏内容，如果有则标记 `⭐ 真题关联`

## 第五步：分析试卷

1. 读取全部试卷，逐题标注考点和分值
2. 统计各考点的出现年份和频率
3. 做趋势判断：新增了什么题型、退出了什么题型
4. 整理成考点频率表

## 第六步：生成复习重点

1. **以老师重点为最高优先级**，与试卷分析结果交叉对比
2. 标出老师重点和真题的重叠区 → P0 必考
3. 标出老师重点但真题未出 → P2 可能
4. 标出老师未提但真题高频 → P1 必须补上
5. 产出文件 `综合复习_重点专题×真题融合版.md`：
   - 按 P0/P1/P2 分级
   - 每个考点含：老师重点内容 + 真题怎么考 + 考试可能怎么问

## 第七步：生成复习策略

1. 产出 `开卷考试_深刻理解vs现场查阅指南.md`（仅开卷）：
   - 分清"必须脑中理解的推理链条"和"可翻资料查的公式/数值"
   - 计算题的方法论、安全性分析的逻辑链 → 必须理解
   - 算法常数、历史年份、S盒表 → 现场可查

2. 产出 `开卷考试_针对性复习计划.md`：
   - 按 3/2/1 天给出时间分配
   - 计算题手算清单 + 简答题口述清单 + 设计题模板

3. 产出 `课件笔记_阅读指南.md`：
   - 标注哪些讲必看（P0），哪些选看（P3）
   - 阅读顺序和时间预估

## 第八步：整理为 Obsidian 知识库

创建如下目录结构：

```
/output/<课程名>/
├── 00-主页.md                   ← 总索引，[[wiki链接]]串联所有文件
├── 01-课件精讲/                  ← 整理版精讲笔记
├── 02-综合复习/                  ← 复习策略+重点专题+真题分析
├── 03-实验笔记/                  ← （如有实验材料）
├── 04-课件原稿/                  ← 原始转换文件（备查）
```

主页需包含：
- 推荐阅读顺序
- 每个文件的摘要说明
- 考点热力图速查表
- 考前 Ctrl+O 快速定位表

每份核心文件的顶部加导航栏：
```
> ⬅️ [[00-主页|← 返回主页]] | [[上一文件|◀ 上一页]] | [[下一文件|▶ 下一页]]
```

## 注意事项

- **全程中文输出**，数学公式用 LaTeX
- 用户偏好简洁直接，反感杂乱输出
- 宁可在精讲中多花时间组织逻辑，避免PPT转换的碎片感
- 所有产出文件统一放在 `/output/<课程名>/` 下
- 若 `/output/<课程名>/` 已存在内容，增量更新而非覆盖
